## Project Structure

```
src/
  routes/          — File-based routes (__root.tsx, index.tsx, about.tsx)
  components/      — Shared components (Header, Footer, ThemeToggle)
  router.tsx       — Router factory
  routeTree.gen.ts — Auto-generated route tree (do not edit)
  styles.css       — Global styles + Tailwind imports
public/            — Static assets
```

## Environment Variables

No environment variables are required. TanStack Start uses `VITE_` prefix for client-exposed env vars. Server-only env vars do not need a prefix but must not be accessed in client code.

## Architecture Decisions

- **File-based routing** via TanStack Router's `@tanstack/router-plugin`.
- **Route tree is auto-generated** — `src/routeTree.gen.ts` is read-only.
- **Tailwind CSS v4** with `@tailwindcss/vite` plugin (no PostCSS config).
- **SSR by default** via TanStack Start's Vite plugin.
- **Devtools** are configured in `__root.tsx` and can be stripped for production via `removeDevtoolsOnBuild` in the Vite plugin.

## Known Gotchas

- `src/routeTree.gen.ts` is auto-generated by `tsr generate`; do not edit manually.
- Tailwind v4 uses `@import "tailwindcss"` in CSS, not a PostCSS config.
- The dev server runs on port 3000 (configured in `package.json` scripts).
- Lucid icons are included but can be removed if unused.
